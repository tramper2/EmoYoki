    - 친근한 느낌의 고품질 3D 일러스트레이션(hero.png) 배치.
    - 버튼에 마우스 호버 마이크로 애니메이션 추가.
- **사용자 경험 개선**: 
    - 테스트 모드 가시성을 위해 상단 알림 바 스타일 조정.
    - 전반적인 컬러 팔레트를 따뜻한 오렌지 계열의 프리즘 스타일로 통일.

## 2. AWS EC2 인프라 구축 및 배포
- **서버 환경**: AWS EC2 t3.small (Ubuntu 24.04), IP: `15.164.212.2`
- **배포 방식**: 
    - 로컬 SSH Key (`Artractive.pem`) 연동.
    - `rsync`를 사용한 소스 코드 동기화 (로컬 -> 서버).
    - `setup_server.sh`를 통한 자동 환경 구성.
- **백엔드 스택**: Docker (FastAPI, PostgreSQL + PostGIS, Redis).
- **프론트엔드 스택**: Vite (React) 빌드 결과물을 Nginx Docker 컨테이너로 서빙.
- **Nginx 리버스 프록시**: 
    - 포트 80 트래픽을 프론트엔드(3000) 및 백엔드 API(8000, `/api` 경로)로 분기.

## 3. 주요 버그 수정 및 최적화 (Troubleshooting)
배포 과정 중 발생한 다음 문제들을 수정하였습니다.

### 백엔드 (FastAPI)
- **스키마 임포트 오류**: `ReviewResponse` 스키마에서 `datetime` 임포트 누락 수정.
- **데이터 타입 상속 오류**: `UserRole`, `UserTier` 등 Enum 클래스에서 중복 상속(`str, str`) 수정.
- **Hatchling 빌드 이슈**: `pyproject.toml`에 패키지 경로(`app`) 명시하여 Docker 빌드 실패 해결.
- **bcrypt 환경 호환성**: 특정 OS/CPU 환경에서 bcrypt 해싱 오류 발생으로 인해 `pbkdf2_sha256` 방식으로 변경하여 안정성 확보.
- **DB 테이블 부재**: 첫 배포 시 DB 테이블이 없어 발생하던 오류를 해결하기 위해 `create_tables.py` 스크립트를 작성하여 자동 테이블 생성 처리.

### 프론트엔드 (React)
- **API URL 하드코딩 해결**: 기존 `http://localhost:8000/api`로 고정된 호출부를 상대 경로(`/api`)로 변경하여 환경에 구애받지 않도록 수정.
- **빌드 파이프라인 최적화**: 배포 시 불필요한 TypeScript 엄격 검증(`tsc`) 단계를 건너뛰도록 `package.json` 수정 (빠른 배포 가능).
