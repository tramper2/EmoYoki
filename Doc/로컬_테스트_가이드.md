# 로컬 테스트 가이드

이모~여기! 서비스를 로컬 환경에서 테스트하는 방법

## 🚀 빠른 시작

### 1. 서비스 실행

```bash
# Docker Compose로 전체 서비스 실행
cd Source
docker-compose up -d

# 백엔드만 실행 (개발용)
cd backend
uvicorn main:app --reload

# 프론트엔드만 실행 (개발용)
cd frontend
npm run dev
```

서비스 접속:
- **프론트엔드**: http://localhost:3000
- **백엔드 API**: http://localhost:8000
- **API 문서**: http://localhost:8000/docs
- **테스트 페이지**: http://localhost:3000/test

---

## 🧪 테스트 페이지 사용법

### 테스트 페이지 접속

1. 브라우저에서 http://localhost:3000/test 접속
2. 다음 기능들을 사용할 수 있습니다:

### 1️⃣ 테스트 계정 생성

**구인자 계정**
- "구인자 계정" 버튼 클릭
- 자동 생성 후 로그인 완료
- 업무 요청 페이지로 이동

**이모님 계정**
- "이모님 계정" 버튼 클릭
- 자동 생성 후 로그인 완료
- 업무 찾기 페이지로 이동

### 2️⃣ 테스트 데이터 생성

"데이터 생성" 버튼 클릭 시 자동 생성되는 데이터:
- **구인자**: 1명 (01000000001 / test1234)
- **이모님**: 5명 (01000000201~01000000601 / test1234)
- **업무**: 10개 (다양한 카테고리)

### 3️⃣ 빠른 로그인

생성된 테스트 계정으로 "빠른 로그인" 버튼 클릭하여 바로 접속

### 4️⃣ 데이터 초기화

테스트가 끝난 후 "데이터 초기화"로 모든 테스트 데이터 삭제

## 🔄 테스트 시나리오

### 시나리오 1: 업무 생성 ~ 완료

```
1. 구인자 계정으로 로그인
2. "새 업무 요청" 클릭
3. 4단계 위저드로 업무 생성
   - 카테고리 선택
   - 지도에서 위치 선택
   - 시간 및 금액 설정
   - 제목 및 설명 입력
4. 업무 등록 완료
```

### 시나리오 2: 이모님 업무 지원

```
1. 이모님 계정으로 로그인
2. "업무 찾기" 페이지 이동
3. 카테고리 필터로 원하는 업무 검색
4. 업무 상세에서 "지원하기" 클릭
5. 메시지 입력 후 지원 완료
```

### 시나리오 3: 매칭 ~ 완료 보고

```
1. 구인자 계정으로 로그인
2. 지원자 확인 후 수락
3. 이모님 계정으로 전환
4. 현장 도착 후 "완료 보고"
   - 완료 사진 URL 입력
   - 완료 메모 작성
5. 구인자 계정으로 전환
6. 완료 확인 후 별점 부여
```

## 🎯 API 테스트

### 직접 API 호출

```bash
# 테스트 계정 생성
curl -X POST http://localhost:8000/test/create-user \
  -H "Content-Type: application/json" \
  -d '{"role": "USER"}'

# 테스트 데이터 생성
curl -X POST http://localhost:8000/test/seed-data

# 테스트 계정 목록 조회
curl http://localhost:8000/test/accounts

# 데이터 초기화
curl -X POST http://localhost:8000/test/reset
```

### Swagger UI 테스트

http://localhost:8000/docs 접속 후:
1. `/test/` 그룹 확장
2. "Try it out" 클릭
3. 파라미터 입력 후 "Execute"

## ⚙️ 설정

### 테스트 모드 활성화/비활성화

**Backend (.env)**
```bash
# 테스트 모드 (인증 우회)
TEST_MODE=true

# 운영 모드 (실제 인증)
TEST_MODE=false
```

**Frontend (.env)**
```bash
# 테스트 모드 표시
VITE_TEST_MODE=true
```

## 🐛 문제 해결

### 1. 데이터베이스 연결 실패
```bash
# Docker 재시작
docker-compose down
docker-compose up -d

# DB 컨테이너 확인
docker ps | grep postgres
```

### 2. 테스트 페이지 접속 안 됨
```bash
# 프론트엔드 포트 확인
# http://localhost:3000 접속 가능한지 확인

# 포트 충돌 시 다른 포트 사용
cd frontend
npm run dev -- --port 3001
```

### 3. 인증이 계속 요청됨
```bash
# .env 파일에서 TEST_MODE=true 확인

# 백엔드 재시작
cd backend
uvicorn main:app --reload
```

### 4. 생성된 계정으로 로그인 안 됨
```bash
# 데이터 초기화 후 다시 생성
curl -X POST http://localhost:8000/test/reset
curl -X POST http://localhost:8000/test/seed-data
```
